Purpose

Project Identity
- Product name: Arch
- Positioning: Fast, compact, mobile-first remake of SRM Academia portal
- Primary audience: SRM students using mobile devices
- Main goals:
  - Reduce friction vs legacy portal
  - Keep login/session reliable for daily use
  - Present attendance and timetable data in compact, high-signal UI
  - Support installable PWA usage

Core Architecture
- Frontend:
  - React 19 + TypeScript + Vite
  - Framer Motion for motion
  - Recharts for marks trend charts
  - Custom CSS design system in one central stylesheet
- Backend:
  - Express server in O:\academis-clone\server\index.cjs
  - Zoho IAM login orchestration using axios + cookie jar
  - Session-token based proxy for SRM pages
- Deployment shape:
  - Frontend on Netlify
  - Backend on Render
  - Netlify rewrites forward /auth/* and /proxy/* to Render

Runtime Model
- Browser logs in via backend (/auth/login).
- Backend stores authenticated SRM cookies in memory by session token.
- Frontend calls backend proxy routes with X-Session-Token.
- Frontend parses returned HTML/JSON into structured app data.

Attendance Notifications and Adaptive Polling
- Notification behavior:
  - Works when permission is granted and app is in standalone PWA mode
  - Detects attendance deltas per course (present, absent, updated)
  - Emits individual notifications (up to 3), then summary if more
- Polling is adaptive (not fixed):
  - active-class: 20s
  - between-classes: 60s
  - off-hours: 7m
  - idle-day: 15m
  - weekend: 25m
  - hidden-tab minimum floor: 3m
- Day-order fetch is throttled to avoid over-fetching:
  - Refresh window: 10 minutes or forced refresh

Auth and Session System

Frontend token/session persistence
- Token storage key:
  - academia.token (sessionStorage or localStorage)
- Session snapshot keys:
  - academia.trusted-session (localStorage)
  - academia.browser-session (sessionStorage)
- Snapshot TTLs:
  - Trusted: 180 days
  - Non-trusted browser session: 2 days

Backend session storage
- In-memory map of sessionToken -> session data
- Session includes cookieString, email, trusted flag, createdAt, lastSeenAt, expiresAt
- Periodic cleanup removes expired sessions

Backend Endpoints (O:\academis-clone\server\index.cjs)
- POST /auth/login
  - Runs Zoho IAM flow and returns sessionToken
- DELETE /auth/sessions
  - Terminates active sessions on upstream
- POST /auth/logout
  - Clears local backend session token
- GET /auth/validate
  - Verifies token and touches TTL
- GET /auth/health
  - Health endpoint with active session count
- /proxy/*
  - Authenticated pass-through proxy to SRM academia pages

Data Fetching and Parsing Layer (O:\academis-clone\src\lib\api.ts)
